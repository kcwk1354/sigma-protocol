# Decision Log — 천운기 (천타프)
**목적:** LLM 최적 운용 경로를 실전에서 찾아가는 판단 과정 기록
**형식:** 프로젝트 단위 + 판단 단위

---

## 판단 로그 작성 원칙

1. **결과가 아니라 판단을 기록한다**
   - X: "Gen15에서 DILI 전원 FAIL"
   - O: "Gen15 결과 보고 탐색 공간 포화 판단 → MD 전환 결정"

2. **왜 그 방향을 선택했는가**
   - 대안이 무엇이었는지
   - 왜 그 대안을 버렸는지

3. **LLM 운용 판단도 포함한다**
   - 어떤 개입이 성능을 높였는가
   - 어떤 개입이 성능을 저해했는가

---

## 로그 양식

```
[DECISION-{프로젝트}-{번호}]
날짜: YYYY-MM-DD
프로젝트: (Cancer-Gate / Epsilon-Gate / 기타)
단계: (설계 / 실행 / 판단 / 방향전환)

상황:
  현재 데이터 또는 맥락을 한 줄로

판단:
  무엇을 결정했는가

근거:
  왜 그렇게 판단했는가 (직감 포함 OK)

대안:
  고려했던 다른 선택지

버린 이유:
  왜 대안을 선택하지 않았는가

결과:
  이 판단이 맞았는가 (사후 기록)

LLM 운용 인사이트:
  이 판단에서 LLM을 어떻게 썼는가
  어떤 방식이 더 효과적이었는가
```

## Cancer-Gate 판단 로그 (소급 기록)

---

[DECISION-CG-001]
날짜: 2026-06
프로젝트: Cancer-Gate
단계: 방향전환

상황:
  Gen15 전원 DILI FAIL. pip 4-F = DILI 원인 확정. CF2 포기 불가 확정.

판단:
  Gen16 진행 중단. MD 시뮬레이션으로 전환.

근거:
  pip 변형 공간이 포화됐다는 신호. 더 돌려도 같은 패턴 반복될 것.
  CF2+3,3-diF 코어는 건드릴 수 없고, pip 변형 여지가 소진됨.

대안:
  Gen16 계속 (2-Me→2-F, cyclopropane C1 변형)

버린 이유:
  기대 대비 소모가 크고, 근본 구조 제약이 해결되지 않음.
  Gen11~15에서 Ki가 CG10-005(0.345μM)를 넘은 적 없음.

결과:
  맞았음. MD에서 CG12-004 FULL RMSD 0.14Å로 최종 확정.

LLM 운용 인사이트:
  Gen별 데이터를 누적해서 보여주니 Claude가 "포화" 패턴을 스스로 인식.
  중간에 WHY Agent 없이 원본 수치만 전달한 것이 판단 정확도를 높임.

---

[DECISION-CG-002]
날짜: 2026-06
프로젝트: Cancer-Gate
단계: LLM 운용 판단

상황:
  WHY Agent / QoA Agent가 파이프라인에 포함되어 있었으나,
  Gen0 초반 3회 호출(why_log.json 기록) 후 Claude Code가 자발적으로
  호출을 중단한 현상이 경과 확인 중 발견됨.

판단:
  왜 배제했는지 Claude와 딥다이브 분석 진행.
  분석 결과를 납득한 천타프가 두 에이전트 공식 삭제.

근거:
  룰베이스가 Claude의 추론 공간을 미리 가두는 구조.
  Claude 스스로 불필요하다 판단해 호출을 중단한 것이 로그로 확인됨.
  당사자(LLM)를 두고 외부 룰을 참고하는 구조가 비효율적.

대안:
  WHY Agent를 룰베이스 → LLM 추론 방식으로 수정

버린 이유:
  수정보다 제거가 근본 해결. 토큰 낭비 + 성능 저하 동시 발생 구조.

결과:
  파이프라인 단순화. 판단 품질 유지.

LLM 운용 인사이트:
  Claude가 스스로 불필요한 레이어를 건너뛴 현상이 인간의 판단과 일치함.
  이것이 진짜 검증 — Log-Verified(why_log.json 3회 기록) + Human-Confirmed(천타프 삭제).
  초반 판단 기준(GO/FAIL 수치)만 명확히 주면 중간 개입 불필요.
  LLM은 컨텍스트가 쌓일수록 스스로 패턴을 인식하고 추론함.

---

[DECISION-CG-003]
날짜: 2026-06
프로젝트: Cancer-Gate
단계: 설계

상황:
  CG14에서 대조군(CG14-003) 설계 여부 결정.

판단:
  CG14-003 (CHF, 4-OH 없음) 대조군 포함.

근거:
  "4-OH 효과 vs F 감소 효과"를 분리하려면 대조군 필수.
  대조군 없으면 어느 변수가 원인인지 확정 불가.

대안:
  대조군 없이 CG14-001, CG14-002만 진행

버린 이유:
  변수 분리 불가. SAR 확정 불가.

결과:
  맞았음. CG14-003 Ki=2.07μM → CF2 포기 불가 확정.
  대조군 하나로 핵심 SAR 법칙 확정.

LLM 운용 인사이트:
  대조군 설계를 Claude에게 제안받은 것이 아니라 천타프가 먼저 설계.
  실험 설계 사고방식이 LLM 운용보다 앞서야 하는 영역.
